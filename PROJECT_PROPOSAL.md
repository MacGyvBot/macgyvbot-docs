## 2. 프로젝트 개요

### 2.1 배경 및 필요성

작업장, 실습실, 작업공간에는 드라이버, 렌치, 플라이어 등 다양한 공구가 반복적으로 사용된다. 그러나 작업 중 필요한 공구를 찾거나 사용 후 다시 제자리에 정리하는 과정은 번거롭고, 공구가 방치되면 분실·파손·안전사고 위험이 커진다. 특히 서랍형 공구 보관함은 공간 활용성이 좋지만, 사용자가 직접 서랍을 열고 필요한 공구를 찾은 뒤 다시 정리해야 하므로 작업 흐름이 자주 끊긴다.

본 프로젝트는 로봇팔, 음성 명령, 카메라 기반 인식 기능을 결합하여 사용자가 자연어로 공구를 요청하면 서랍에서 해당 공구를 꺼내 전달하고, 사용 후에는 다시 지정 위치에 정리하는 공구 관리 어시스턴트를 구현하고자 한다. 이를 통해 단순 픽앤플레이스(pick-and-place)가 아니라 사람의 작업을 보조하는 인터랙티브 로봇 서비스를 프로토타이핑한다.

### 2.2 프로젝트 목적

사용자의 음성 명령을 인식하여 서랍에 보관된 공구 중 필요한 공구를 찾아 사용자 앞 전달 위치로 제공하고, 반납된 공구를 다시 서랍의 지정 위치에 정리하는 로봇팔 기반 공구 관리 어시스턴트를 구현한다.

### 2.3 프로젝트 범위

**포함(In-Scope)**

- 제한된 공구 3~4종 인식 및 요청 처리: 예) 십자드라이버, 일자드라이버, 렌치, 플라이어
- 사용자의 음성 명령에서 공구 이름 추출: 예) “십자드라이버 가져다줘”, “렌치 줘”
- 단일 서랍 또는 모형 서랍의 열기/닫기 동작 구현
- 서랍 내부의 공구별 지정 위치 기반 파지 및 전달
- 사용자 손 직접 전달 대신, 1차 목표에서는 사용자 앞 고정 전달 트레이에 공구 배치
- 사용 후 반납 트레이에 놓인 공구를 원래 위치로 재정리

**제외(Out-of-Scope)**

- 날카롭거나 무거운 위험 공구의 실제 사용자 손 전달
- 움직이는 사용자 손을 실시간 추적하여 직접 쥐여주는 기능
- 공구가 심하게 겹쳐 있거나 가려진 상황에서의 완전한 인식
- 산업 현장 수준의 안전 인증 및 고속 자동화 성능

## 3. 최종 산출물(Deliverables)

- 동작 가능한 로봇팔 기반 공구 전달 및 정리 시스템(하드웨어 + 소프트웨어)
- 음성 명령 처리, 공구 선택, 로봇팔 제어를 포함한 통합 소스코드 및 README
- 서랍 또는 모형 서랍, 전달 트레이, 반납 트레이를 포함한 시연 환경
- 공구 인식 및 위치 추정을 위한 테스트 이미지 데이터셋
- 3분 내외 시연 영상: 음성 요청 → 서랍 열기 → 공구 전달 → 반납 → 재정리
- 최종 보고서 및 발표 자료

## 5. 주요 접근 방법 / 아이디어

하드웨어는 6축 로봇팔, 그리퍼, 상방 고정형 카메라, 단일 서랍 또는 모형 서랍, 전달 트레이, 반납 트레이로 구성한다. 초기 단계에서 공구는 정리함에 위치시키고, 로봇팔은 공구 파지와 전달 동작에 집중한다. 확장 단계에서는 로봇팔이 직접 서랍 손잡이를 잡고 여닫으며 서랍에 공구를 정리하는 방식도 고려한다.

소프트웨어는 음성 명령 입력, 공구 이름 추출, 공구 위치 결정, 로봇팔 동작 계획, 상태 표시 모듈로 구성한다. 음성 명령은 STT 결과에서 공구 키워드를 매칭하는 방식으로 시작하며, 공구 위치는 초기에는 사전 좌표 기반으로 처리한다. 이후 카메라 기반 공구 인식과 위치 보정을 추가하여 공구가 약간 틀어진 상황에서도 대응하도록 확장한다.

전체 시스템은 실패 위험을 줄이기 위해 큰 작업부터 시작해 점차 세부 기능을 추가하는 단계적 개발 전략을 따른다. 최소 기능만 완성되어도 “요청된 공구를 가져다주는 로봇 어시스턴트”로 시연할 수 있고, 추가 단계가 성공할수록 서랍 개폐, 재정리, 방향 보정, 공구 상태 확인 등으로 완성도를 높일 수 있다.

### 5.1 단계별 구현 전략

| 단계 | 목표 | 성공 기준 | 비고 |
| --- | --- | --- | --- |
| 1단계 | 지정 위치의 공구를 집어 전달 위치로 이동 | 좌표가 고정된 공구를 집어 전달 트레이에 내려놓기 | 음성·서랍 없이 핵심 픽앤플레이스 검증 |
| 2단계 | 공구 종류별 다른 위치에서 가져오기 | 입력된 공구명에 따라 올바른 좌표 선택 | 터미널 입력 또는 버튼 입력 가능 |
| 3단계 | 사용 후 공구 재정리 | 지정된 정리 위치에 공구 정리 | 사용자 손 직접 회수는 제외 |
| 4단계 | 서랍 개폐 및 공구 파지 | 공구 전달 전 서랍 열기, 작업 후 서랍 닫기 | - |
| 5단계 | 음성 명령으로 공구 선택 | 명령 문장에서 공구 이름을 추출하고 해당 공구 선택 | 키워드 매칭 기반으로 시작 |
| 6단계 | 사용자 손 방향으로 공구 전달 | 사용자가 직접 파지 후 그리퍼 릴리즈 | 확장 목표 |

### 5.2 시스템 동작 흐름

1. 사용자가 음성으로 공구 요청: “렌치 가져다줘”
2. 음성 인식 결과에서 공구 이름 추출
3. 해당 공구의 서랍 열기 동작 수행
4. 요청 공구의 위치 확인 또는 사전 좌표 조회
5. 로봇팔이 공구를 파지
6. 사용자에게 공구 전달 및 반납 공구 회수
7. 로봇팔이 반납 공구를 원래 위치로 정리
8. 서랍 닫기 동작 수행

## 6. 기대 효과 및 활용 방안

- 로봇팔 제어, 음성 인터페이스, 객체 인식, 좌표 변환, 시스템 통합을 한 프로젝트에서 경험할 수 있다.
- 단순 분류 로봇이 아니라 사용자의 요청을 이해하고 작업을 보조하는 어시스턴트형 로봇 서비스를 구현한다.
- 실습실, 공구실, 메이커스페이스 등에서 공구 관리 보조 시스템으로 확장 가능성을 보여줄 수 있다.
- 시연 과정이 직관적이다. “공구 요청 → 서랍 열림 → 공구 전달 → 반납 → 정리” 흐름을 관람자가 쉽게 이해할 수 있다.
- 기능을 단계적으로 축소·확장할 수 있어 실패 위험을 관리하면서도 최종 완성도를 높일 수 있다.

## 8. 핵심 차별점

본 프로젝트는 로봇팔이 단순히 물체를 집어 옮기는 것을 넘어, 사용자의 음성 요청을 받아 필요한 공구를 찾아 전달하고 사용 후 다시 정리하는 상호작용형 로봇 어시스턴트를 목표로 한다. 또한 서랍이라는 실제 보관 환경을 포함하여 공구 관리의 전체 흐름을 다룬다는 점에서 일반적인 픽앤플레이스 프로젝트와 차별화된다.

특히 단계적 구현 전략을 통해 최소 기능인 공구 전달만으로도 시연 가능한 결과물을 확보하고, 추가 구현 단계에 따라 서랍 개폐, 공구 재정리, 위치 보정, 공구 상태 확인 등으로 확장할 수 있다. 따라서 실패 위험을 줄이면서도 프로젝트 완성도와 발표 효과를 동시에 확보할 수 있다.
